## Project Intent

Build a standalone Streamlit app (`app.py`) + development Jupyter notebook (`label_generator.ipynb`) that:

1. Accepts a CSV of photographic paper sample data.
2. Generates diamond-shaped radar chart glyphs (two formats: with and without gridlines).
3. Produces formatted PDF label sheets for printing on 8.5 × 11" paper.

The app will be deployed to users via `pip install -r requirements.txt && streamlit run app.py` — no Jupyter or VS Code required.

## Key Design Decisions

1. **Glyph axis inversion preserved**: GlossUnits is inverted so high gloss → matte axis points inward (same as reference code).
2. **Glyph crop**: Values >1.0 (outside normalization range) produce a larger diamond; the PDF clips it to the 6.5cm square boundary via ReportLab `clipPath`.
3. **Key always bottom-right**: The slot iteration skips (col=2, row=3) for data labels; `draw_key_cell` is called independently.
4. **Year page-break**: `build_pages` groups by Year before chunking into LABELS_PER_PAGE=11 groups. Partial pages padded with `None`.
5. **Font**: Aptos registered as TTFont if found at `C:\Windows\Fonts\Aptos.ttf`; otherwise falls back to Helvetica (ReportLab built-in, visually equivalent to Arial).
6. **NaN handling — numeric fields**: If a value is NaN, the field displays without a units suffix (e.g., "Texture: NaN" not "Texture: NaN Sq").
7. **NaN handling — Backprint**: Renders as "none" (not "NaN") since absence of backprint text is meaningful.
8. **NaN handling — glyph data**: If 1 of the 4 glyph axes is missing, that axis collapses to 0.0 (the diamond becomes a triangle). If 3 or more axes are missing, the glyph is skipped entirely (`diamond_path = None`).
9. **PDF naming**: `Labels_MMDDYYYY.pdf` (e.g., `Labels_04282026.pdf`).
10. **Glyph overwrite**: `generate_glyphs()` uses `shutil.rmtree` to delete and recreate both output folders on every run, ensuring stale PNGs from previous runs are never retained.
11. **Input persistence**: Working directory and output folder are saved to `label_generator_config.json` (next to `app.py`) after each successful run and reloaded automatically on Streamlit startup.
12. **Brand warning suppressed**: Missing Brand values are common and not actionable; only missing Manufacturer triggers a warning.
13. **Key image via file uploader**: The Streamlit sidebar includes an optional file uploader for the key image. If provided, it is written to `{working_dir}/key.png` before PDF assembly. If omitted, the bottom-right cell is left blank.
